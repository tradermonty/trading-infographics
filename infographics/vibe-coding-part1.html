Vibe Coding を最大限に活用する方法

作成日: 2024年5月29日
情報源: "How To Get The Most Out Of Vibe Coding"
および "バイブコーディングツール解説"

このドキュメントは、YCのパートナーであるTom氏および創業者たちによる、「Vibe Coding」（AIコーディングツールを活用したプログラミング手法）を効果的に実践するためのアドバイスとベストプラクティスをまとめたものです。AIを単なるコード生成ツールとしてではなく、新しい種類のプログラミング言語や共同作業者として捉えることが成功の鍵となります。

Vibe Codingとは何か？

「AIを新しいタイプのプログラミング言語として考えよう。コードではなく言語でプログラミングする」

Vibe Codingは、従来のコードではなく「言語」を使ってプログラミングを行う新しい手法です。

良い結果を得るためには、必要なコンテキストと情報を非常に詳細に提供する必要があります。📌

計画と段階的な実装

1
LLMと協力して包括的な計画を立てる
2
計画をMarkdownファイルに保存し、常に参照する
3
セクションごとに実装し、各ステップが機能することを確認
「セクション2だけをいま実装しよう、機能することを確認して、コミットしてから計画に戻り、セクション2を完了としてマークしよう」
「pure LLMでスコープと実際のアーキテクチャを構築するのに不合理なほどの時間を費やすことが非常に重要です」

バージョン管理 (Git) の徹底

Gitを熱心に使用し、各ステップの実装後にはコミットします。

✅ 新機能に取り組む前に「clean git slate」から始める
✅ AIが「vision quest」に入った場合は git reset head hard で復帰
「バージョン管理はあなたの味方です。Gitを信仰的に使いましょう」

テスト主導のアプローチ

1
まずテストケースを作成（LLMを使用せず手作業で）
2
ユニットテストより高レベルの統合テストを推奨
「サイトやアプリをクリックして機能がエンドツーエンドで動作することをシミュレーションしましょう」
LLMには無関係なロジックに変更を加える悪癖があります。テストスイートでリグレッションを早期に捕捉しましょう！🔍

バグ修正のアプローチ

1
エラーメッセージをそのままLLMにコピー＆ペースト
2
複雑なバグは、LLMに原因を3〜4つ考えさせる
3
修正試行ごとにgit resetで新たに開始
「バグに遭遇したら、まずエラーメッセージをそのままLLMにコピペするのが最初のステップです」
「バグ修正の試みが複数回失敗した場合、リセットせずに繰り返さないでください。なぜなら、LLMはただゴミの層を重ねていくだけだからです」

指示ファイルの活用

各ツール専用の指示ファイルを作成
Cursor Rules, Windsurf Rules など
数百行に及ぶ詳細な指示を記述
指示ファイルは、AIコーディングエージェントの効果を大幅に向上させます！📝

ドキュメントへのアクセス

1
関連APIのドキュメントをダウンロード
2
作業フォルダのサブディレクトリに配置
3
指示ファイルで「実装前にドキュメントを読む」と明示
LLMエージェントのオンラインウェブドキュメントへのアクセスはまだ不安定な場合があります

LLMを教師として活用

「不慣れなプログラミング言語や技術について学ぶのにLLMを教師として活用できます」

実装したコードをLLMに一行ずつ解説させることで、理解を深めることができます。👨‍🏫

Stack Overflowをスクロールするよりも効果的な学習ができます

複雑な機能の実装

✅ 複雑な新機能はスタンドアロンで先に実装
✅ 参照実装があれば、ダウンロードしてLLMに指示
✅ その後、大規模コードベースに再実装
パート1: 基本戦略 パート2: 高度なテクニックとツール
